# VERITAS — Content Verification System

> Multi-agent AI fact-checker. Paste any text, get a sourced confidence report for every claim inside it.

Built for the **Band of Agents Hackathon** (June 2026).

---

## What It Does

VERITAS takes a block of text, extracts every verifiable factual claim from it, dispatches parallel research agents to investigate each claim across the web and academic sources, and returns a structured report with a per-claim confidence score, verdict, and cited sources.

**Input** → a paragraph, article snippet, or short note  
**Output** → a scored, sourced verification report

---

## Features

- **Automatic claim extraction** — separates verifiable facts from opinions using an LLM
- **Parallel research pipeline** — 4 independent researcher agents run simultaneously
- **Academic + web coverage** — general web search (Tavily) and academic-domain search (PubMed, arXiv, DergiPark, etc.)
- **Source validation** — a dedicated agent cross-checks findings from all researchers before the report is written
- **Plain-text report** — final report generated by Gemma 4 26B (Featherless), human-readable
- **Rate limiting** — 5 requests / IP / day, thread-safe
- **Security hardened** — HTML sanitization on input, X-Frame-Options, X-Content-Type-Options, Referrer-Policy headers

---

## Architecture Overview

```
Flask API  →  Gateway Agent  →  Band chat room  →  7 specialist agents
```

All 8 agents communicate exclusively through a shared Band SDK chat room. No direct in-process calls between agents. See [AGENTS.md](AGENTS.md) for the full pipeline breakdown.

## Quick Start

### 1. Prerequisites

- Python 3.10+
- A `.env` file with all API keys (copy from `.env.example`)

### 2. Install dependencies

```bash
python -m venv .venv
.venv\Scripts\activate        # Windows
source .venv/bin/activate     # macOS / Linux

pip install -r requirements.txt
```

### 3. Configure environment

```bash
cp .env.example .env
# Fill in all keys: BAND_*, AIML_API_KEY, TAVILY_API_KEY, FEATHERLESS_API_KEY
```

Each of the 8 agents needs its own `*_AGENT_ID` and `*_API_KEY` from the Band dashboard. The `BAND_CHAT_ROOM_ID` must be a fresh room when onboarding new agents.

### 4. Run the backend

```bash
.venv\Scripts\python.exe server\app.py
# Server starts on http://localhost:5000
# All 8 agents start automatically in background threads
```

### 5. Open the frontend

```bash
start frontend\index.html
# Or serve statically:
.venv\Scripts\python.exe preview_server.py   # → http://localhost:3456
```

## API

### `POST /api/analyze`

```json
// Request
{ "text": "Your claim text here." }

// Response
{
  "task_id": "abc12345",
  "report": {
    "text": "TASK_ID: ...\n\nCLAIMS:\n1. ...\n\nAVERAGE SCORE: 85/100\n\nRESULTS:\n..."
  },
  "raw": "<full Band message>"
}
```

Rate limit: **5 requests per IP per day** (resets at midnight UTC). Returns `429` when exceeded.

### `GET /api/health`

```json
{ "status": "ok", "service": "VERITAS" }
```
